变频双驱动器，新版本软件变更说明：
时间：2021/06/03		版本：V1.2			作者：LHC

新增/修改：
1、修改了密码控件触发方式，默认无密码，进入系统后修改密码生效。
2、增加了系统参数存储到IAP_FLASH区的CRC校验方式，增加系统稳定性。
3、修改了串口驱动，并加入了多串口通讯模型。
4、修改定时器底层驱动。
5、修改了变频器数据存储PLC方式。由原来的逐条存取，变换为块地址连续存储。

工作流程：
①首先串口3通过485协议定时（1S）向变频器请求数据，最终变频器回复的数据将会暂存到RS485缓冲区；
②定时对接收数据通过ModbusRTU协议处理（每条数据200ms）后，通过串口4（STC与STM32直连串口）发送给PLC固定区域存储；
③云端1min钟发送一条请求命令向PLC询问数据，串口2（4G模块）通过多串口协议获得相应数据。

特别说明：
①串口屏幕（迪文屏）直连PLC串口；PLC中STM32程序是固化程序，无法修改。
②屏幕数据呈现方式和实际读数据方式相反；显示速度较慢原因有：屏幕本身问题、PLC取出数据速度、PLC数据存入速度等。

量产无线PLC(昆明理工大学学生用)，新版本软件变更说明：
时间：2021/12/15		版本：V1.3			作者：LHC

新增/修改：
1、彻底优化了系统参数的存储结构。
2、大幅度修改了菜单构造框架。
3、增加了无线模块启停菜单（4G/wifi）。
4、增加了wifi模块的AP热点启用/停用菜单。
5、为每个PLC增加唯一ID（与APID同名）。
6、恢复出厂设置中加入了wifi模块复位程序。
7、全新的多串口转发通讯模型（效率更高、通信更加稳定）。

关于PLC系统烧录程序特别说明：
① 通过串口烧录器(CH340)对STM32F103VET6烧录文件“224XP_STM32VET6_V5.36”。
（BOOT0端子首先需要拉低；烧录软件必须使用“STMFlashLoader Demo”，在点击“next”之前最好把芯片“rest”引脚拉到低电平进行
复位，否则会出现软件卡死情况）
② 通过8PIN座子烧录器对M95512存储器芯片烧录文件“224XP_25AA512_V5.36”。
（烧录特别注意点： 烧录系统在维修部门张经理xp系统，烧录芯片类型为：M95512（ISP）,实测其他几个类型选项可以烧录，但是
PLC系统无法工作；烧录芯片有图标一面朝上摆放；如果是二次烧录芯片，请做引脚清洗操作，否则有可能提示引脚接触不良）
③ 组装顺序：操作流程必须是先对stm32芯片进行烧录完成后，再焊接M95512芯片。（如果直接焊接好烧录好的M95512芯片，则在擦除stm32芯片
时存储器M95512可能会被擦除，造成PLC无法使用，此时可以通过拆除当前存储芯片，更换新的烧录好的存储芯片解决，原拆机芯片必须重新烧录后方可使用）

其他说明：
1、物联网模块的单个节点数小于20个字节后会导致部分节点失去响应（Frame_Flag不满足）。
2、串口通讯模型分为主从结构，stc的串口1、 2、3通过串口4对PLC的port0进行分时复用的访问。
3、当串口1、2、3作为主机时，通过串口4与PLC构成的网络为从机。
4、当串口1、2、3作为从机时，PLC作为主机，由屏幕设置stc串口4与其他口的通讯。
